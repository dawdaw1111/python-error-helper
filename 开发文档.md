# PyErr（Python报错速查）MVP文档

## 一、项目概述

### 项目名称

PyErr（Python报错速查）

### 一句话定位

输入 Python 报错信息，快速返回错误原因、排查步骤和解决方案。

### 核心价值

- 不用再到处搜索报错
- 快速定位问题原因
- 提供标准化排查流程
- 降低初学者学习成本

------

## 二、目标用户

### 主要用户

- Python 初学者
- 编程培训学员
- 刚入门开发的工程师

### 用户痛点

- 看不懂报错
- 不知道从哪排查
- 搜索效率低
- 解决问题耗时长

------

## 三、MVP目标

验证三个核心点：

1. 用户是否愿意粘贴报错使用工具
2. 规则库是否能覆盖高频错误
3. 用户是否能通过结果解决问题

------

## 四、核心功能（MVP范围）

### 1. 报错输入

支持：

- 单行错误
- 完整 traceback

示例：

```
ModuleNotFoundError: No module named 'requests'
```

------

### 2. 错误匹配引擎

匹配方式：

- 正则匹配（优先）
- 精确关键词
- 包含匹配

支持错误类型：

- ModuleNotFoundError
- SyntaxError
- TypeError
- NameError
- IndexError
- KeyError
- AttributeError 等

------

### 3. 结果展示页

统一结构：

#### 错误名称

ModuleNotFoundError：找不到模块

#### 错误解释

Python 在导入模块时未找到对应包。

#### 常见原因

- 依赖未安装
- 虚拟环境错误
- 模块名拼写错误

#### 排查步骤

1. pip show requests
2. 检查解释器
3. 检查虚拟环境

#### 解决方案

```
pip install requests
```

#### 相关推荐

- ImportError
- pip安装无效问题

------

### 4. 搜索功能

支持：

- 中文搜索（如：模块找不到）
- 英文错误名
- 模糊匹配

------

### 5. 后台规则管理

功能：

- 新增规则
- 编辑规则
- 删除规则
- 查看命中率

------

### 6. 用户反馈

每条结果提供：

- 👍 有帮助
- 👎 没帮助

用于优化规则库。

## 七、匹配逻辑

流程：

1. 提取错误类型（如 TypeError）
2. 优先匹配 regex
3. 再匹配关键词
4. fallback 到通用错误解释
